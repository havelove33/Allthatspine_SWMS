## 1. 시스템 개요

### 1.1 목적
대표(관리자)가 **원격지에서** 직원의 근무·업무·목표 이행을 통제하고, 회사의 프로젝트 현황과 예산 현황을 모니터링하기 위한 웹 기반 시스템.

해결하려는 문제:
- 직원 출근이 불규칙한데 원격이라 체계적 관리가 불가능
- 관리자 부재로 직원 업무 수행 점검이 어려움
- 프로젝트 추진 현황이 한눈에 파악되지 않음
- 예산 관리가 체계적으로 이루어지지 않음

### 1.2 핵심 철학 — "모니터링·통제 도구" (NOT ERP)

> 본 시스템은 회사의 업무를 **운영(처리)**하는 무거운 ERP가 아니라, 대표가 원격에서 상황을 **모니터링·통제**하는 가벼운 도구다.

| 만들지 않는 것 (ERP식, 제외) | 만드는 것 (모니터링식) |
|---|---|
| 정식 회계 처리 · 대사 · 세무 신고 | 매출/지출 현황 입력 + 잔액 조회 |
| 풀 전자결재 엔진 (전결 · 대결 · 결재선) | 간단 신청 → 승인 |
| 풀 프로젝트 관리 (간트 · 의존성 · WBS) | 프로젝트 진행률 · 현황 조회 |
| 인사관리 (급여 · 평가 · 발령) | 시스템 구동에 필요한 최소 직원정보 |

**설계 원칙: 입력은 최소로, 보는 것은 충실하게.**

### 1.3 대상 사용자
- 전 직원 (출퇴근·업무보고·미션·게시판 등)
- 회계 담당자 (예산 입력)
- 대표/관리자 (전체 모니터링·승인·설정)

### 1.4 지원 환경
- **반응형 웹** (PC · 모바일 동시 지원, 단일 시스템)
- 향후 **PWA** 확장 가능 (홈 화면 추가 · 푸시 알림)
- PC 브라우저 / 모바일 브라우저 동일 주소 접속

## 2. 기술 스택

| 구분 | 기술 | 역할 |
|------|------|------|
| 호스팅 | **Vercel** | 앱 배포 (서버리스) |
| 백엔드 | **Supabase** | Database(PostgreSQL) · Auth(인증) · Storage(파일) · Realtime |
| 보안 | Supabase **RLS** (Row Level Security) | 행 단위 접근 통제 (예산 격리, 개인정보 보호) |
| 프론트엔드 | (미정, Next.js 유력) | 화면 · 로직 |

> Vercel은 서버리스라 앱 서버에 파일을 직접 저장할 수 없음 → 모든 파일은 Supabase Storage에 저장하고, DB에는 링크·권한·메타데이터만 보관.

## 3. 사용자 역할 및 권한

| 역할 | 설명 | 주요 권한 |
|------|------|-----------|
| **직원** | 일반 직원 | 본인 근태·보고·미션 작성, 게시판, 본인 통계 조회 |
| **회계담당** | 예산 입력 담당 | 직원 권한 + 예산 모듈 입력/수정 |
| **부서장** | (확장용, 1단계 미사용) | 직원 권한 + 소속 직원 조회 |
| **대표/관리자** | 최고 관리자 | 전체 조회·승인·설정·통계, 예산 전체 조회 |

- 권한은 Supabase Auth + RLS로 구현
- 예산 모듈은 **회계담당·대표만** 접근 (그 외 메뉴 자체 미표시)
- 민감정보(인사·예산)는 RLS로 DB 레벨 차단

## 5. 모듈별 기능 명세

> 모듈 번호는 **개발 우선순위 순**으로 정렬 (5.1~5.4 = MVP 핵심).

### 5.1 인사정보 (HR) — 기반 모듈

**목적**: 모든 모듈의 출발점이 되는 직원 기본 데이터 관리. (인사관리 시스템이 아닌 최소 정보)

**주요 기능**
- 직원 등록/수정: 이름, 사번, 연락처, 이메일, 프로필 사진
- 입사일 (→ 연차 자동계산 기준), 재직상태 (재직/휴직/퇴사)
- 직급·직책, 고용형태 (정규/계약/인턴)
- 부서 필드 (1단계 미사용, 확장 대비 데이터만 확보)
- 역할 지정: 직원 / 회계담당 / 부서장 / 대표 (Auth·RLS 연결)
- 계정 활성/비활성, 비밀번호 초기화
- 퇴사 처리: 계정 비활성화 + 기록 보존 (근태·보고 이력 유지)

**권한**: 본인 + 대표(+인사담당) 열람. 등록/수정은 대표.

**라이트화 (제외)**: 급여, 인사평가, 발령, 경력관리 등 인사관리 기능 전부 제외.

---

### 5.2 근태관리 — 직원 통제 핵심

**목적**: 원격 환경에서 직원의 출퇴근을 위치 기반으로 통제하고, 근태 현황을 실시간 파악.

**주요 기능**

*출퇴근 체크*
- 출근/퇴근 원클릭, **서버 시각 기준** 저장 (PC 시간 조작 방지)
- 기록 항목: 출근시각, 퇴근시각, 근무시간 자동계산, 기록 IP, 접속 기기
- 기록 누락 시 정정 요청 → 대표 승인/반려
- 외근/재택 예외 모드 (대표 승인제, 단순 처리)

*위치 인증 (우회 차단)*
- **1차 — 회사 IP 대역(/24)**: 유동 IP 대응 위해 대표가 대역 직접 추가/수정 가능
- **2차 — QR 스캔**: 사무실 부착 QR (1단계는 고정 QR, 추후 TOTP 회전 코드로 강화)
- **2차 대안 — GPS 지오펜싱**: 회사 좌표 반경 내 (모바일)
- PC: IP + QR / 모바일: GPS + QR
- 목적: 원격 데스크톱 우회 접속을 QR/GPS로 차단

*휴가·연차*
- 휴가 신청/승인 워크플로 (→ 전자결재·캘린더 연동)
- 종류: 연차, 반차(오전/오후), 반반차, 병가, 경조사, 공가, 무급
- 입사일 기준 연차 자동발생·잔여 계산
- 휴가일은 결근/지각 통계에서 자동 제외
- 공휴일·주말 자동 인식

*통계*
- 지표: 결근, 지각, 조기퇴근, 정상출근, 야근, 평균 출근시각, 총 근무시간
- **월별 기본 뷰** (캘린더 색상 표시 + 우측 통계 패널)
- **연간 통계** (별도 메뉴, 월별 추이 그래프)
- 대상: 개인별 / 회사 전체
- 엑셀 내보내기

*관리자 기능*
- **실시간 출근 현황판** (누가 출근/미출근 — 원격 관리 핵심 화면)
- 미체크 직원 알림
- 근무 규칙 설정 (기준시각, 지각 인정범위, 코어타임, IP 대역 관리)

**라이트화**: QR은 고정 QR부터 시작, TOTP 회전은 후순위.

### 5.3 업무보고 — 직원 통제 핵심

**목적**: 직원이 정해진 템플릿에 맞춰 일/주/월 업무를 보고하게 하여, 무엇을 했는지 체계적으로 파악.

**주요 기능**

*보고 체계*
- 일일 / 주간 / 월간 보고
- 일→주→월 **자동 취합** (※ 2단계 구현, 1단계는 독립 작성)
- 마감 시간(예: 매일 18시) 미작성자 자동 리마인더

*템플릿 (서술식 방지)*
- 항목(필드) 기반 강제 입력 — 필수 항목 미작성 시 제출 불가
- **관리자가 보고 유형별 항목을 직접 추가·수정·삭제** (커스터마이즈)
- 항목 타입: 텍스트 / 선택 / 다중선택 / 진행률 / 시간(숫자) / 프로젝트태그 / 리스트
- 기본 일일 템플릿 예: 업무제목, 연관 프로젝트(필수), 업무구분, 진행상태, 진행률, 오늘 한 일(리스트), 소요시간, 이슈

*프로젝트 태그*
- 관리자가 프로젝트 태그 마스터 등록 → 직원이 보고 작성 시 선택(필수)
- 태그로 프로젝트·미션·통계와 연결

*관리자 업무관리 페이지*
- **미제출 현황판** (오늘/이번 주 미작성자)
- 프로젝트별 통계 (투입 인원·총 공수·진행률)
- 보고별 코멘트 + 직원 답글 + 읽음 표시
- 개인별 업무 히스토리 타임라인
- 키워드/프로젝트 검색, 이슈 에스컬레이션
- (선택) AI 요약: 전체 일일보고 → 요약 + 주의 이슈

### 5.4 미션 (목표관리) — 직원 통제 핵심

**목적**: 개인별 주/월 목표를 협의 설정하고, 실제 업무가 목표에 부합하는지 점검.

**주요 기능**

*미션 작성 (협의)*
- 직원 제안 / 관리자 지시 / 공동 편집 모두 가능
- 상태: 작성 → 승인 → 진행 → 완료/평가 (단순화)
- 작성·수정 주체 이력 보존

*미션 템플릿*
- 제목, 기간(주간/월간), 프로젝트 태그, **측정가능 목표지표**, 우선순위, 달성기준, 진행률
- "측정 가능한 목표" 강제 (지표·달성기준 필수)

*목표 부합 체크 (핵심)*
- 업무보고 작성 시 해당 미션 선택해 연결
- 미션별 진행 현황 자동 집계
- 진행률: 보고 연동 **자동 + 수동 수정** 병행
- **미션 외 업무**: '기타'로 허용하되 **비율 표시** (목표 이탈 감지)
- 주말/월말 자동 리뷰 (달성률 vs 투입시간 대조)

*미션 평가*
- 기간 종료 시 자기평가 + 관리자평가 (달성/미달/초과)
- 미달 사유 → 다음 기간 반영
- 개인 목표 달성률 추이 누적

**라이트화**: 복잡한 협의 상태머신 제거, 작성→승인→진행으로 단순화.

### 5.5 프로젝트 — 모니터링 모듈

**목적**: 프로젝트를 이해(개요)하고 추진 현황을 한눈에 모니터링. (프로젝트 관리 도구가 아닌 현황 조회)

**주요 기능**

*개요 (블로그/위키형)*
- 리치 에디터: 서술 + 이미지·표·파일·링크
- 추진 목표 / 배경 / 범위 / 기대효과 / 전략
- 수정 이력(버전), 되돌리기

*세부 업무 (체크리스트형)*
- 담당자 / 기한 / 상태(예정·진행·완료·보류) / 진행률
- 업무보고 프로젝트 태그와 자동 연결 → 보고 작성 시 현황 갱신

*추진 일정 (마일스톤 타임라인)*
- 주요 이정표 시간순 표시, D-day·지연 표시

*연계 하위 프로젝트*
- 상위–하위 트리, 하위 진행률 상위 롤업

*추진현황 대시보드*
- 전체 진행률 / 일정 준수율 / 투입 인원·공수 / 남은 업무
- 관련 업무보고·미션 자동 수집
- **신호등 상태**: 정상(초록)/지연(노랑)/위험(빨강)
- 전체 프로젝트 포트폴리오 뷰
- 활동 피드, 댓글, 파일 자료실, 알림, 검색·필터

**라이트화 (제외)**: 간트차트, 프로젝트 간 의존관계, 복잡한 자동 롤업 계산, 풀 WBS.

### 5.6 예산관리 🔒 — 모니터링 모듈 (접근 통제) *(라이트화)*

**목적**: 회계담당이 통장·법인카드의 자금 입출금을 입력하고, 대표가 잔액·사용 현황을 추적 모니터링. (회계 시스템이 아니라 **"자금 현황판"**)

> ⚠️ **범위(라이트화)**: 매출·매입 손익, 부가세, 정부과제 분리트랙, 증빙 대사, 계정과목별 집행률 등은 **제외**.
> 실제 범위 = **통장 입출금 내역 · 계정별 잔액 · 법인카드 사용내역의 "현황 조회"** 뿐. ("입력 최소, 보기 충실")

**접근 통제**: 회계담당(입력/수정) + 대표(조회)만. 일반 직원 메뉴 미표시.

**주요 기능**

*자금 계정 등록*
- 통장 / 법인카드 / 현금을 계정으로 등록, 각 계정의 **기초 잔액**을 1회 입력
- 예: 기업은행 운영통장, 신한 법인카드

*통장 입출금 내역*
- 회계담당이 입금/출금 거래를 일자·금액·적요·거래처로 기입
- 계정별 **잔액 자동 추정** = 기초잔액 + Σ입금 − Σ출금

*법인카드 사용내역*
- 카드 계정의 결제(지출) 내역 + 적요 + 거래처
- 카드별·월별 사용액 집계

*자금 현황 대시보드*
- 통장 **잔액 합계** + 이달 카드 사용액 요약(현황 카드)
- 거래내역 테이블 (월·계정·구분 필터)

**라이트화 (제외)**: 매출/매입 손익·부가세, 수금/미수 추적, 정부과제 비목 관리, 증빙 자동대사, 예산편성 대비 집행률, 세무자료 내보내기, 사전 지출결의 승인. (필요 시 별도 검토)

---

### 5.7 게시판 — 보조 모듈

**목적**: 사내 공지 전달 및 직원 간 소통.

**주요 기능**
- **공지사항 + 자유게시판 통합** (카테고리로 구분)
- 공지: 관리자 작성, **로그인 시 팝업 표시 옵션** (오늘 하루 보지 않기 / 다시 보지 않기)
- 공지: 필독 지정, 읽음 확인(수신 명단), 상단 고정, 예약 발행
- 자유: 전 직원 작성, 댓글·대댓글, 이모지 반응, @멘션
- (선택) 익명 건의함 — 대표만 열람
- 리치 에디터, 파일 첨부, 검색·정렬, 신고·블라인드, 알림 연동

---

### 5.8 자료공유 — 보조 모듈

**목적**: 사규·양식·매뉴얼 등 공용 자료 보관·공유.

**주요 기능**
- 폴더 구조 (카테고리)
- 파일 업로드/다운로드 (Supabase Storage, DB는 링크·권한 관리)
- **대용량 파일은 외부 링크(구글 드라이브 등) 등록 병행**
- 권한: 전체 공개 / 특정 인원만
- 검색·정렬, 미리보기(이미지·PDF), 버전(최신본)

> 예산·정부과제 증빙은 자료공유에 노출되지 않고 예산 모듈 내에서만 접근 (접근통제 유지).

---

### 5.9 일정·캘린더 — 보조 모듈

**목적**: 각 모듈에 흩어진 날짜를 한 곳에 모아 보는 통합 뷰.

**주요 기능**
- 월/주/일 보기 + 리스트 보기
- **자동 수집**: 근태(휴가) / 프로젝트(마일스톤·기한) / 미션(기간) / 공지(행사)
- 일정 유형: 개인 / 회사 / 프로젝트 / 미팅
- 공유 캘린더 vs 개인 캘린더 토글
- **부재 현황** (누가 언제 휴가인지)
- 일정 알림, 공휴일 자동 표시

**라이트화**: 풀 일정관리(회의실 예약 등) 제외, "모아 보는 뷰" 중심.

---

### 5.10 전자결재 — 보조 모듈

**목적**: 휴가·지출 등 승인이 필요한 행위의 간단한 신청→승인 처리.

**주요 기능**
- **간단 신청 → 승인** 구조 (결재선·전결·대결 엔진 없음)
- 양식: 휴가신청 / 지출신청 / 구매요청 / 일반 기안
- 상태: 대기 / 승인 / 반려 / 회수
- 반려 사유 필수, 코멘트
- 신청 알림 (대표에게), 처리 지연 리마인더
- 대표 모바일 승인함 (어디서든 승인/반려)
- 첨부파일(증빙), 감사 이력
- 연동: 휴가신청 승인 시 → 근태·캘린더 자동 반영

**라이트화 (제외)**: 다단계 결재선, 전결 규정, 대결, 금액별 자동 결재선.

---

## 6. 모듈 간 연동 (한 번 입력, 여러 곳 반영)

```
                  ┌─────────── 프로젝트 ───────────┐
                  │ 개요·세부업무·일정·현황          │
                  └──────────────┬─────────────────┘
                     프로젝트 태그 (양방향 연결)
        ┌──────────────────┬─────┴────────┬──────────────────┐
     업무보고             미션            근태               예산
   "뭘 했나"          "목표대로?"      "언제·누가"        "돈 현황"
        │                  │               │
        └──────────────────┴───────────────┴──→ 캘린더 (날짜 집계)
                           │
                      전자결재 (휴가·지출 승인 → 근태·예산·캘린더 반영)
                           │
                      인사정보 (전 모듈의 직원·권한·연차 기반)
```

**핵심**: 직원이 일일보고에 프로젝트 태그 + 소요시간을 입력하면 → 프로젝트 현황 + 미션 달성률 + 개인 통계에 동시 반영.

---

## 7. 개발 로드맵

```
■ 1단계 (MVP) — "직원 통제부터 작동시킨다"
  ① 인사정보(기반) → ② 근태 → ③ 업무보고 → ④ 미션
  └ "누가 언제 일하고, 뭘 했고, 목표대로 하나"가 완성

■ 2단계 — "모니터링 추가"
  ⑤ 프로젝트 현황 → ⑥ 예산 현황

■ 3단계 — "보조 편의"
  ⑦ 게시판 → ⑧ 자료공유 → ⑨ 캘린더 → ⑩ 전자결재
```

1단계만 완성해도 "원격 직원 통제"는 작동. 이후 모니터링·편의 기능을 단계적으로 확장.

## (부록) v1.0 종합 — 추가·완성 기능 (2026-06-05)

v1.1 이후 구현되어 **v1.0.0** 릴리스에 포함된 기능 종합. (시스템명: **올댓스파인 스마트 업무관리 시스템 / SWMS**)

### A. 통합 대시보드 = 역할별 종합 상황판
메뉴 나열을 없애고, 로그인 시 **역할별 상황판**을 제공.
- **대표(관리자)**: KPI(오늘 출근·일일보고 제출률·결재 대기·주의 프로젝트) + **처리 대기함**(휴가/근태정정/전자결재/나의업무 승인 건수를 한곳에) + 위젯(오늘 근무 현황·프로젝트 신호등[세부업무 펼침]·보고 미제출자·다가오는 일정·예산·최근 공지).
- **직원**: 내 오늘(출퇴근·보고·미션·결재) + **해야 할 일(To-do)** + 미션 진행률 + 이번달 근태 + 다가오는 일정 + 최근 공지.
- **회계담당**: 직원 상황판 + 예산 현황 위젯.

### B. 예산 — 매출 목표 추적
- 연도별 **매출 항목 + 연간 목표(필수)·월간 목표(선택)** 설정.
- 예산 화면 최상단 **매출 현황**: 연간 총 매출 **목표 달성률**, **세부 항목별 매출 비중·달성률**, **월별 매출 추이 그래프**.
- 입금 거래에 매출 항목을 지정하면 실적 자동 집계(라이트 모델: 매출 = 입금).

### C. 데이터 백업/복원 (신규)
- 중요 데이터 손실 대비. 전 테이블 + 계정을 **스냅샷(JSON)** 으로 백업.
- **3중 보호**: 매일 자동(클라우드) + 수동(클라우드+다운로드) + **로그인 시 로컬 자동 다운로드**(관리자 대시보드 스위치, **기본 OFF**).
- 복원: 관리자 전용, "복원" 확인 입력, **merge 방식**(기존 행 삭제 없음, auth 계정 제외).

### D. 디자인·브랜딩
- 시스템명 **"스마트 업무관리 시스템(SWMS)"** 확정, 헤더 중앙 타이틀.
- 브랜드 틸-블루 테마 + **Pretendard** 기본 폰트.
- 척추 + AI 네트워크 추상 모티프: **로그인 풀스크린 배경**(PC/모바일), **대시보드 히어로 배너**, **빈 상태 일러스트**.
- 금액 입력 **천 단위 콤마** 자동, 링크 공유용 **OG 이미지**.
